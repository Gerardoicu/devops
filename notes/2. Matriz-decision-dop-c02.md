# Matriz de Decision DOP-C02

Objetivo:

- resolver preguntas largas con reglas cortas
- reducir el espacio de opciones rapido
- memorizar decisiones, no historias

Como usar este archivo:

1. Busca la palabra o restriccion dominante.
2. Encuentra la fila mas cercana.
3. Parte de esa decision.
4. Valida el detalle fino en las opciones.

Importante:

- los IDs banco de cada fila comparten familia de decision
- no significan que todas las preguntas tengan la misma respuesta literal
- usa la fila para arrancar, no para contestar en automatico

## 1. Compliance continuo

Si ves:

- `NON_COMPLIANT`
- regla de `AWS Config`
- enforcement continuo
- tags obligatorios
- logging obligatorio

Empieza pensando en:

- `AWS Config`
- remediacion con `SSM Automation` o accion administrada si el escenario pide enforcement o autoremediacion

No confundas con:

- `EventBridge` de creacion puntual
- Lambda custom si ya existe remediacion administrada

Detalle fino que decide:

- recurso exacto
- regla exacta
- estado exacto

- 7, 12, 15, 18, 21, 32, 60, 70, 89, 103, 108, 117, 119

## Checklist de uso rapido

Antes de marcar una opcion, confirma:

1. servicio correcto
2. feature correcta dentro del servicio
3. recurso correcto
4. estado o evento correcto
5. capa correcta para filtrar o automatizar
